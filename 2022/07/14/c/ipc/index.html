无名管道

管道通常用于具有亲缘关系的父子进程或者兄弟进程间通信，是半双工的，数据只能往一个方向流动，先入先出。要想双向通信，需要建立两个管道。

pipe创建的管道两端处于一个进程中，用于父子进程间的通信。管道建立时会创建两个文件描述符，pipefd[0]固定用于读管道，而pipefd[1]固定用于写管道，构成一个半双工的通道。

如果读一个写端已被关闭的管道，在所有数据都被读取后，read返回0指示达到了文件结束处。
如果写一个读端已被关闭的管道，则产生信号SIGPIPE。如果忽略该信号或者捕捉该信号并从其处理程序返回，则write出错返回，errno设置为EPIPE。

标准流管道

popen提供连接到另一个进程的标准输入或标准输出的一个单行管道，创建的管道必须使用标准I/O函数进行操作。popen先执行fork,然后调用exec以执行command，并且返回一个标准I/O文件指针。如果type是”r”，则文件指针连接到command的标准输出。如果type是”w”，则文件指针连接到command的标准输入。

pclose用来关闭创建的管道流，返回由popen所执行的进程的终止状态。

命名管道

命名管道在非父子、兄弟进程之间也可相互通信。FIFO遵循先进先出规则，对FIFO的读总是从开始处返回数据，写则把数据添加到末尾。

mkfifo创建一个FIFO管道，对于读进程。若该管道是阻塞打开，且当前FIFO内没有数据，则对读进程而言将一直阻塞到有数据写入。若该管道是非阻塞打开，则不论FIFO 内是否有数据，读进程都会立即执行读操作。即如果FIFO内没有数据，则读函数将立刻返回 0。

对于写进程。若该管道是阻塞打开，则写操作将一直阻塞到数据可以被写入。若该管道是非阻塞打开而不能写入全部数据，则读操作进行部分写入或者调用失败。

  • O_RDONLY 读管道
  • O_WRONLY 写管道
  • O_RDWR 读写管道
  • O_NONBLOCK 非阻塞
  • O_CREAT 如果该文件不存在，那么就创建一个新的文件，并用第三个参数为其设置权限。
  • O_EXCL 如果使用O_CREAT时文件存在，那么可返回错误消息。可测试文件是否存在。
返回错误类型 说明
EACCESS 所指定的目录路径无可执行的权限
EEXIST 所指定的文件已存在
ENAMETOOLONG 的路径名称太长
ENOENT 包含的目录不存在
ENOSPC 文件系统的剩余空间不足
ENOTDIR 路径中的目录存在但却非真正的目录
EROFS 指定的文件存在于只读文件系统内

信号

号是软件中断，用于在多个进程之间传递异步信号。信号可以直接进行用户空间进程和内核进程之间的交互。kill -l命令可以查看系统支持的信号列表。

  • SIGABRT 调用abort函数时产生此信号。进程异常终止。
  • SIGALRM 超过用alarm函数设置的时间时产生此信号。由setitimer函数设置的间隔时间已经过时。
  • SIGBUS 指示一个实现定义的硬件故障。
  • SIGCHLD 在一个进程终止或停止时被送给其父进程。
  • SIGCONT 此作业控制信号送给需要继续运行的处于停止状态的进程。如果接收到此信号的进程处于停止状态,则系统默认动作是使该进程继续运行，否则默认动作是忽略此信号。
  • SIGEMT 指示一个实现定义的硬件故障。
  • SIGFPE 此信号表示一个算术运算异常，例如除以0，浮点溢出等。
  • SIGHUP 如果终端界面检测到一个连接断开，则将此信号送给与该终端相关的控制进程。
  • SIGILL 此信号指示进程已执行一条非法硬件指令。
  • SIGINT 当用户按中断键(Ctrl+C)时，此信号送至前台进程组中的每一个进程。
  • SIGIO 此信号指示一个异步I/O事件。
  • SIGIOT 这指示一个实现定义的硬件故障。
  • SIGKILL 杀死任一进程。不能被捕捉或忽略。
  • SIGPIPE 如果在读进程已终止时写管道，则产生此信号。当套接口的一端已经终止时，若进程写该套接口也产生此信号。
  • SIGPOLL 在一个可轮询设备上发生一特定事件时产生此信号。
  • SIGPROF 当setitimer函数设置的梗概统计间隔时间已经超过时产生此信号。
  • SIGPWR 它主要用于具有不间断电源(UPS)的系统上。蓄电池电压过低时将信号发送给init进程，然后由init处理停机操作。
  • SIGQUIT 当用户在终端上按退出键(Ctrl+)时，产生此信号，并送至前台进程组中的所有进程。终止前台进程组并产生core文件。
  • SIGSEGV 指示进程进行了一次无效的存储访问。
  • SIGSTOP 这是一个作业控制信号，它停止一个进程。不能被捕捉或忽略。
  • SIGSYS 指示一个无效的系统调用。
  • SIGTERM kill命令发送的系统默认终止信号。
  • SIGTRAP 指示一个实现定义的硬件故障。
  • SIGTSTP 交互停止信号，当用户在终端上按挂起键(Ctrl+Z)时，终端驱动程序产生此信号。
  • SIGTTIN 当一个后台进程组进程试图读其控制终端时，终端驱动程序产生此信号。当读进程忽略或阻塞此信号，或读进程所属的进程组是孤儿进程组时不产生此信号，读操作出错并设置errno。
  • SIGTTOU 当一个后台进程组进程试图写其控制终端时产生此信号。当写进程忽略或阻塞此信号，或写进程所属的进程组是孤儿进程组时不产生此信号，写操作出错并设置errno。
  • SIGURG 此信号通知进程已经发生一个紧急情况。在网络连接上，接到非规定波特率的数据时，此信号可选择地产生。
  • SIGUSR1 这是一个用户定义的信号，可用于应用程序。
  • SIGUSR2 这是一个用户定义的信号，可用于应用程序。
  • SIGVTALRM 当一个由setitimer函数设置的虚拟间隔时间已经超过时产生此信号。
  • SIGWINCH 如果一个进程用ioctl设置窗口大小命令更改了窗口大小，则内核将信号送至前台进程组。
  • SIGXCPU 如果进程超过了其软CPU时间限制，则产生此信号。
  • SIGXFSZ 如果进程超过了其软文件长度限制产生此信号。

signal

signal设置信号的处理方法，成功返回handler的值,若出错则为SIG_ERR。对于handler参数：

  • SIG_IGN 表示忽略此信号。(SIGKILL和SIGSTOP不能忽略)
  • SIG_DFL 表示执行系统默认动作。
  • 自定义的信号处理函数指针。

信号处理函数应调用可重入的函数。

raise kill

raise给进程本身发送信号。

kill给一个进程发送信号，pid参数:

  • pid>0 将信号发送给进程ID为pid的进程。
  • pid==0 将信号发送给与其进程组ID相等的进程。
  • pid<0 将信号发送给进程组ID等于pid绝对值的进程。

进程将信号发送给其他进程需要许可权。对于非root用户，其规则是发送者的实际或有效用户ID必须等于接收者的实际或有效用户ID。

alarm

alarm函数可以设置一个闹钟时间,超时后产生SIGALRM信号。如果不忽略或不捕捉此信号，则其默认动作是终止进程。

每个进程只能有一个闹钟时间。如果在调用alarm时，以前已设置过闹钟时间，而且还没有超时，则该闹钟时间的余留值作为本次alarm函数调用的值返回。以前登记的闹钟时间则被新值代换。如果以前登记的尚未超过的闹钟时间，而且seconds值是0，则取消以前的闹钟时间，其余留值仍作为函数的返回值。

pause

只有执行了一个信号处理程序并从其返回时，pause才返回。在这种情况下，pause返回-1，errno设置为EINTR。

信号量

信号量用于进程同步，只有获得了信号量的进程才可以运行，没有获得信号量的进程则只能等待。

信号量包括一个称为信号量的变量和在该信号量下等待资源的进程等待队列，以及对信号量进行的两个原子操作(PV操作)。其中信号量对应于某一种资源，取一个非负的整型值，信号量值指的是当前可用的该资源的数量，若它等于 0 则意味着目前没有可用的资源。PV原子操作的具体定义如下:

  • P操作：获得信号量，将值-1。如果值非负，表明有可用的资源，则占用一个资源直接返回，否则被阻塞，直到别的任务释放该信号量才能运行。它会导致调用的进程睡眠，因此不能在中断上下文使用。

  • V操作：释放信号量，把值+1，如果值为非正数，表明有任务等待该信号量，因此唤醒这些任务。

semget创建信号量或获得在系统已存在的信号量，函数返回信号量标识符。

key参数是信号量的键值，不同进程可通过使用同一个信号量键值来获得同一个信号量。其中有个特殊值 IPC_PRIVATE。它用于创建当前进程的私有信号量。

nsems参数是需要创建的信号量数目。

semflg参数同open函数的权限位，可以用八进制表示，使用IPC_CREAT 标志创建新的信号量，即使该信号量已经存在也不会出错。如果同时使用 IPC_EXCL 标志则创建一个新的的信号量，如果该信号量已经存在，该函数会返回出错。

semctl初始化信号量。semid参数是信号量标识符。semnum参数是信号量编号,当使用信号量集时才会被用到。值为0就是单个信号量。

cmd参数指定对信号量的操作, 若要传递参数，必须自定义semun联合体。对于单个信号量:

  • IPC_STAT 获得该信号量的semid_ds结构,并存放在第4个参数arg的buf中。
  • IPC_SETVAL 将信号量值设置为arg中的val值。
  • IPC_GETVAL 返回信号量的当前值
  • IPC_RMID 删除信号量

对于信号量集合：

  • GETVAL 返回semnum的信号量値。
  • SETVAL 设置semnum的信号量値。该值由a rg . v a l指定。
  • GETPID 返回semnum的信号量的sempid值。
  • GETNCNT 返回semnum的信号量的semncnt值。
  • GETZCNT 返回semnum的信号量的semzcnt值。
  • GETALL 获取集合中所有信号量的值，并由arg.array返回。
  • SETALL 按arg.array指向的数组设置集合中所有信号量。

semop进行信号量的PV操作，若成功返回信号量标识符。操作已经被删除的信号量会出错。

sops指向一个sembuf对象。其中sem_op是信号量操作:取值为-1，则表示P操作；取值为1，则表示V操作。sem_flg通常设置为SEM_UNDO，在进程没释放信号量而退出时，系统自动释放该进程中未释放的信号量。

nsops是操作数组sops中的操作个数。

共享内存

共享内存允许两个或多个进程共享一给定的存储区，因为数据不需要在客户机和服务器之间复制，所以效率很高，常用于不同进程间进行大量数据传递。内核为每个共享存储段设置了一个shmid_ds结构。ipcs -m命令可以查看系统的共享内存。共享存储段紧靠在栈之下。

shmget创建共享内存，成功返回共享内存段标识符。

key参数是共享内存的键值，多个进程可以通过它访问同一个共享内存,其中有个特殊值 IPC_PRIVATE。它用于创建当前进程的私有共享内存。

size参数是共享内存区大小。shmflg参数同open函数的权限位，也可以用八进制表示法。

shmat映射共享内存，把这段创建的共享内存映射到具体的进程空间中。成功返回被映射的段地址，出错返回-1。

shmaddr参数是共享内存映射的目的地址，若为0则表示系统自动分配地址。

shmflg参数默认为0，表示共享内存可读写，若为SHM_RDONLY，则共享内存只读。

shmdt撤销映射的地址。

shmctl函数对共享存储段执行多种操作。cmd参数:

  • IPC_STAT 对此段取shmid_ds结构,并存放在buf指向的结构中。
  • IPC_SET 按buf指向的结构中的值设置与此段相关结构中的下列三个字段:shm_perm.uid、shm_perm.gid以及shm_perm.mode。此命令只能由下列两种进程执行：一种是其有效用户ID等于shm_perm.cuid或shm_perm.uid的进程;另一种是具有root特权的进程。
  • IPC_RMID 删除共享存储段。删除后不能再用shmat与该段连接。此命令只能由下列两种进程执行：一种是其有效用户ID等于shm_perm.cuid或shm_perm.uid的进程;另一种是具有root特权的进程。
  • SHM_LOCK 锁住共享存储段。此命令只能由root执行。
  • SHM_UNLOCK 解锁共享存储段。此命令只能由root执行。

/dev/zero的可映射为共享内存。设备/dev/zero在读时,是0字节的无限资源。此设备也接收写向它的任何数据，但忽略此数据。

mmap根据文件描述符创建一个存储区，其长度是参数length，将其取整为系统上的最近页长。存储区都初始化为0。flags指定MAP_SHARED标志，则此存储区可共享。

消息队列

消息队列是消息的链接表，存放在内核中并由消息队列标识符标识，用户可以从消息队列中添加消息和读取消息等。消息队列具有一定的FIFO特性，但是也可以按消息的类型字段取消息，每个消息包含一个正长整型类型字段，一个非负长度以及实际数据字节，每个消息队列都有一个msqid_ds结构与其相关，规定了队列的当前状态。

msg-first和msg-last指针分别指向相应消息在内核中的存放位置，它们对用户进程
而言是无价效的。

msgget创建或打开消息队列，这里创建的消息队列的数量会受到系统消息队列数量的限制。成功返回消息队列id。

key参数是消息队列的键值，多个进程可以通过它访问同一个消息队列，其中有个特殊值 IPC_PRIVATE，用于创建当前进程的私有消息队列。

msgflg参数是权限标志位。

msgsnd把消息添加到已打开的消息队列末尾。
msgp指向消息缓冲区，该消息结构msgbuf通常为:

msgsz参数是消息正文的字节数(不包括消息类型指针变量)。msgflg参数为IPC_NOWAIT，则不阻塞；为0则阻塞直到发送成功为止。

msgrcv把消息从消息队列中取走，与FIFO不同的是，这里可以指定取走某一条消息。msgp指向消息缓冲区，msgsz是消息正文的字节数。
msgtyp参数：

  • 0 接收消息队列中第一个消息。
  • 大于0 接收消息队列中第一个类型为msgtyp的消息。
  • 小于0 接收消息队列中第一个类型值大于msgtyp绝对值且类型值最小的消息。

msgflg参数：

  • IPC_NOWAIT 则不阻塞。
  • 0 则阻塞直到发送成功为止。
  • MSG_NOERROR 若返回的消息比msgsz字节多，则截短到msgsz字节。

消息发送端进程和消息接收端进程之间不需要实现进程间的同步。一般发送端发送的消息类型设置为该进程的进程号，因此接收端根据消息类型确定消息发送者的进程号。

msgctl用来控制消息队列。cmd命令参数：

  • IPC_STAT 读取消息队列的数据结构 msqid_ds，并将其存储在buf中。
  • IPC_SET 设置消息队列的数据结构 msqid_ds中的msg_perm的值(IPC操作权限)。
  • IPC_RMID 从系统内核中删除消息队列。
