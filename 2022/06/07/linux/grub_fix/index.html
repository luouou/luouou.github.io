linux修复grub引导

发表于 2022-06-07 更新于 2022-06-08 分类于 linux 阅读次数：

因为之前给硬盘分区的原故，使磁盘盘符发生了变化，进而导致grub无法引导系统，出现以下错误：

在救援模式下，可以使用以下命令：

  • set 设置环境变量
  • ls 查看目录
  • insmod 加载模块
  • root 指定用于启动系统的分区，即安装了grub的分区
  • prefix 设定grub配置文件和启动路径

先使用 ls 命令可以列出当前的所有分区，我们的目的是要找到grub所在的分区，可以使用ls (hd0, msdos6)/boot/grub这样的形式一个个分区地支探测，能成功的列出文件内容的就是grub分区。

找到正确的分区后，接着就是用正确的分区设置环境变量，然后加载模块。

执行完成后就能成功进入系统了，因为只是手动进入了系统，其实grub并没有真正修复，下次启动同样会出错。所以进入系统后需要安装Boot-Repair工具，完成grub的修复。

还有种情况就是grub已经损坏，已经无法进入系统了，需要重新安装grub。

先通过linux安装光盘进入系统，使用fdisk -l命令查看磁盘，将boot分区挂载到系统上。

安装完后，如果重启grub的引导界面上没有windows系统，则需要再更新下grub引导项
